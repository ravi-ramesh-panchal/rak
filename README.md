## Student Fee Collection Microservices

This repo contains two Spring Boot 3 microservices demonstrating student management and fee collection with H2, Spring Data JPA.

- **student-service**: manages student master data
- **fee-service**: collects fees and generates receipts, validating students via `student-service`

### 1. Technology stack

- **Spring Boot** 3.2.x (REST, Spring Web, Spring Data JPA)
- **H2** in-memory databases
- **springdoc-openapi** for Swagger UI
- **JUnit 5** for basic tests

### 2. Running the services

Both services are independent Maven projects under this folder.

#### 2.1 Start student-service

From a terminal:

```bash
cd student-service
mvn spring-boot:run
```

The service will start on **port 8081**.

- H2 console: `http://localhost:8081/h2-console`
- Swagger UI: `http://localhost:8081/swagger-ui.html`
- OpenAPI JSON: `http://localhost:8081/v3/api-docs`

#### 2.2 Start fee-service

In another terminal:

```bash
cd fee-service
mvn spring-boot:run
```

The service will start on **port 8082**.

- H2 console: `http://localhost:8082/h2-console`
- Swagger UI: `http://localhost:8082/swagger-ui.html`
- OpenAPI JSON: `http://localhost:8082/v3/api-docs`

> Note: `fee-service` uses the property `student.service.base-url=http://localhost:8081` (configured in its `application.properties`) and calls `student-service` before creating a receipt.

### 3. REST APIs

#### 3.1 student-service APIs

Base URL: `http://localhost:8081`

- **POST** `/api/students`
  - **Body**:
    ```json
    {
      "studentId": "STU-001",
      "name": "StudOne",
      "grade": "5",
      "mobileNumber": "9876543210",
      "schoolName": "Dubai school"
    }
    ```
  - **Response**: `201 Created` with created student.

- **GET** `/api/students/{studentId}`
  - Example: `/api/students/STU-001`
  - **Response**: `200 OK` with student data or `404` if not found.

- **GET** `/api/students`
  - **Response**: list of all students.

#### 3.2 fee-service APIs

Base URL: `http://localhost:8082`

- **POST** `/api/receipts`
  - Validates student with `student-service`.
  - **Body**:
    ```json
    {
      "studentId": "STU-001",
      "amount": 1500.0,
      "paymentMode": "CASH",
      "description": "Tuition fee for March"
    }
    ```
  - **Response**: `201 Created` with generated receipt (including `receiptNumber` and `paymentDate`).

- **GET** `/api/receipts/{id}`
  - **Response**: single receipt or `404` if not found.

- **GET** `/api/receipts`
  - Optional query: `?studentId=STU-001` to filter by student.

### 4. Swagger / OpenAPI

Swagger UI and OpenAPI JSON are automatically generated by **springdoc-openapi**:

- **student-service**
  - Swagger UI: `http://localhost:8081/swagger-ui.html`
  - OpenAPI spec: `http://localhost:8081/v3/api-docs`
- **fee-service**
  - Swagger UI: `http://localhost:8082/swagger-ui.html`
  - OpenAPI spec: `http://localhost:8082/v3/api-docs`

You can export these JSON specs directly from the browser or via `curl` if you need static files.

### 5. Postman collection

A Postman collection is provided at:

- `StudentFeeMicroservices.postman_collection.json`

Import this file into Postman to get ready-made requests for:

- Creating and listing students
- Collecting fees and listing receipts (with and without `studentId` filter)

### 6. Error handling & resilience

- Both services have `@RestControllerAdvice` with simple JSON error responses.
- `fee-service` uses a `WebClient` with a timeout to call `student-service`. Failures to validate the student (down service, timeouts) are translated into a `502 Bad Gateway` with a readable message.

### 7. Tests

Each service includes a minimal `@SpringBootTest` to verify that the Spring context starts without errors. You can run them with:

```bash
cd student-service
mvn test

cd ../fee-service
mvn test
```

This completes the basic functional and documentation requirements for the student fee collection microservices.
